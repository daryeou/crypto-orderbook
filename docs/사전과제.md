```
DEADLINE:      4일
STACK:         Kotlin, Jetpack Compose
ARCHITECTURE:  자유 (선택 근거 기술)
```

---

## Overview

암호화폐 거래소의 공개 API를 활용하여 **종목 리스트**와 **실시간 호가창**을 구현하는 Android 앱을 만듭니다.

**평가 항목**:

- 코드 설계와 아키텍처 판단력
- WebSocket 기반 실시간 데이터 처리 능력
- AI 도구 활용 하여 업무 프로세스로 활용하는 방식

**평가하지 않는 것**:

- 픽셀 퍼펙트한 UI 디자인
- 암호화폐 도메인 지식
- 전체 기능의 완성도

이 과제는 대규모 앱을 완성하는 것이 목적이 아닙니다. 핵심 기능이 동작하는 수준의 결과물과, 그 과정을 기록한 문서를 기대합니다. 요구사항에 해당하는 기능중 미구현 항목은 ROADMAP.md에 기술합니다.

## Requirements

### Must-have

아래 세 가지 기능을 구현합니다.

**1. 종목 리스트**

거래소 REST API에서 코인 목록을 조회하여 다음 정보를 표시합니다:
- 종목명
- 현재가
- 24h 변동률

**2. 호가창 (Order Book)**

종목 리스트에서 코인을 선택하면 해당 종목의 호가창을 표시합니다:
- 매수/매도 호가를 시각적으로 구분
- 각 호가의 가격과 수량 표시
- ask, bid 호가 사이에 직전 체결가 혹은 거래소가 제공하는 현재가를 표시

**3. 실시간 갱신**

호가창 데이터는 WebSocket을 통해 실시간으로 갱신합니다. REST polling은 허용하지 않습니다.

### Nice-to-have

아래 항목은 선택 사항입니다. 구현 시 가점을 부여합니다.

- **에러 처리**: 네트워크 에러, WebSocket 연결 끊김 시 재연결, 로딩 상태, 빈 상태 UI
- **테스트 코드**: 범위와 전략은 자유. 미작성 시 ROADMAP.md에 테스트 전략 기술
- 기타 본인 판단으로 추가한 기능

## AI 도구 활용

이 과제에서 **모든 AI 도구의 사용을 허용합니다**.

단, AI 도구를 사용이 필수는 아니며, 만약 AI 사용시 활용 과정을 기록한 아래의 템플릿에 해당하는 **RETROSPECTIVE.md** 문서, 혹은 자체적으로 작성한 바이브 코딩 회고 문서가 필요합니다.

### 기본 RETROSPECTIVE.md 템플릿

아래 구조를 참고합니다. 형식은 자유이나, 각 항목의 내용들이 다른 형태로라도 포함되는것을 권장합니다.

```markdown
# 바이브 코딩 회고

## 사용 도구
(도구 이름, 모델/버전)

## 작업 전략
- 문제 분해 방식
- AI 활용 영역 vs 직접 작성 영역
- 전체 작업 흐름

## 구체적 AI 상호작용 사례 (3~5개)

각 사례마다 아래 항목 포함:

### 사례 1: (제목)
- **프롬프트**: (요약 또는 원문)
- **AI 출력**: (요약)
- **판단**: 수락 / 수정 / 거부
- **이유**: (판단 근거)
- **수정 내용**: (수정한 경우)

### 사례 2: ...

## AI가 만든 실수
- 발견한 문제와 감지 방법

## 아키텍처 결정
- 본인이 직접 내린 설계 판단
- AI 제안을 거부한 경우 그 이유

## 솔직한 평가
- AI가 효과적이었던 부분
- AI의 한계를 느낀 부분
- 다음에 다르게 할 점
```

## Evaluation Criteria

| 영역 | 비중 | 핵심 평가 포인트 |
|------|------|------------------|
| 코드 품질 & 아키텍처 | 20% | 레이어 분리, 의존성 방향, Compose 상태 관리, 코드 가독성 |
| WebSocket 실시간 처리 | 25% | 연결 관리, 데이터 스트림 처리, 라이프사이클 연동 |
| 구현 및 검증 프로세스 | 35% | 문제 정의부터 구현까지의 방식, 만들어진 결과에 대한 최종 검증 방식 |
| UI/UX(디자인X) & 가점 기능 | 20% | 상태 처리 (로딩/에러/빈), 테스트, 추가 기능 |
